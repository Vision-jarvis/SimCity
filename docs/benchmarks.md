# Benchmarks & Empirical Validation

This document is the reproduction guide for every number in `main.tex` and
`results/FINDINGS.md`. All experiments run on CPU; a full pass is a few hours.

## TL;DR of the findings

1. **Headline (replicated on synthetic AND real data):** SimCity's
   narrative-conditioned Hawkes excitation predicts **which narratives will
   transfer across platforms** — a task where any static Hawkes process is
   random (AUC 0.5) *by construction*, because it has one global α for all
   narratives.
   - Synthetic benchmark: AUC **0.653 ± 0.005** (3 seeds, Mann-Whitney
     p < 1.1e-4, survives popularity- and reach-stratification).
   - Live HN+GDELT corpus: AUC **0.632** (p = 0.0012, above the popularity
     baseline 0.612).
2. **Honest negative result:** raw engagement-magnitude regression is
   noise-dominated under near-critical cascade dynamics. Even an *oracle*
   handed the true excitation feature has negative residual skill (−0.107) and
   near-random surge-ranking AUC (0.507). Don't headline MAE on this task.
3. **Supporting:** next-platform timing AUC 0.610 ± 0.005 vs static MHP
   0.605 — above the baseline on every seed but not significant at n=3
   (p = 0.23).

Full narrative with every intermediate experiment: [`results/FINDINGS.md`](../results/FINDINGS.md).

## Datasets

### Synthetic multiplex benchmark (controlled ground truth)

Events are generated by a per-narrative, cross-platform Hawkes branching
process: each narrative has a latent bridge factor (controls Reddit→HN→News
transfer) and a latent reach factor; branching is clamped near-critical so the
target is ~68% within-narrative (temporal) variance.

```bash
python data/synthetic_generator.py --events 10000 --narratives 320 --out data/synthetic_events.pkl
```

### Real multiplex corpus (live Hacker News + GDELT)

Keyless collector; each run **accumulates** into a deduplicated raw store
(`data/real_events_raw.pkl`), so the corpus grows across runs:

```bash
python -m data.build_real_dataset --pages 2 --gdelt-topics 45 --gdelt-timespan 1w --out data/real_events.pkl
```

`scripts/accumulate_real_data.cmd` wraps this for Windows Task Scheduler
(a daily task named `SimCity_accumulate` keeps growing the corpus; more data
tightens the real-data confidence intervals).

## Training

`train.py` is controlled by environment variables:

| Env var | Meaning | Benchmark setting |
|---|---|---|
| `SIMCITY_DATA` | dataset path | `data/synthetic_events.pkl` / `data/real_events.pkl` |
| `SIMCITY_EPOCHS` | training epochs | 15 |
| `SIMCITY_SEED` | torch/numpy seed | 1, 2, 3 |
| `SIMCITY_TAG` | suffix for prediction dumps | `seed1`, `real`, ... |
| `SIMCITY_TGN_W` / `SIMCITY_HAWKES_W` / `SIMCITY_VIRALITY_W` | task-loss weights | 0.1 / 10 / 0.1 (Hawkes-weighted variant) |
| `SIMCITY_EXC_DIM` | excitation-conditioned virality head (0 disables) | 0 |

The Hawkes-weighted setting matters: with uniform weights the multi-task loss
dilutes the Hawkes head below the static baseline (documented in the paper's
limitations).

```bash
# one benchmark run (per seed)
SIMCITY_EPOCHS=15 SIMCITY_TGN_W=0.1 SIMCITY_HAWKES_W=10 SIMCITY_VIRALITY_W=0.1 \
SIMCITY_SEED=1 SIMCITY_TAG=seed1 python train.py
```

Each run writes `results/simcity_metrics.json` plus per-event test dumps
`results/simcity_test_preds_<tag>.npz` (virality preds, per-narrative bridge
scores `alpha_off`) and `..._<tag>_hawkes.npz` (causal per-event intensities).

## Full reproduction, in order

```bash
# 1. data
python data/synthetic_generator.py --events 10000 --narratives 320 --out data/synthetic_events.pkl

# 2. three seeds of the Hawkes-weighted model
for s in 1 2 3; do
  SIMCITY_EPOCHS=15 SIMCITY_TGN_W=0.1 SIMCITY_HAWKES_W=10 SIMCITY_VIRALITY_W=0.1 \
  SIMCITY_SEED=$s SIMCITY_TAG=seed$s python train.py
done

# 3. baselines
python -m baselines.run_virality_baselines --epochs 15
python run_hawkes_baseline.py --data data/synthetic_events.pkl --epochs 8 --out baselines/hawkes_report.json

# 4. evaluations
python multiseed_stats.py --seeds 1 2 3
python narrative_transfer_eval.py --preds results/simcity_test_preds_seed2.npz
python run_benchmarks.py
python compute_residual_metrics.py
python burst_ranking_eval.py
python analysis_tools/oracle_residual_check.py

# 5. real data (repeat over days to grow the corpus)
python -m data.build_real_dataset --pages 2 --gdelt-topics 45 --gdelt-timespan 1w --out data/real_events.pkl
SIMCITY_DATA=data/real_events.pkl SIMCITY_EPOCHS=15 SIMCITY_TGN_W=0.1 \
SIMCITY_HAWKES_W=10 SIMCITY_VIRALITY_W=0.1 SIMCITY_TAG=real python train.py
python narrative_transfer_eval.py --preds results/simcity_test_preds_real.npz --out results/narrative_transfer_real.md
python platform_prediction_eval.py --data data/real_events.pkl

# 6. paper figure
python scripts/make_results_figure.py
```

## Sanity controls built into the pipeline

- **Chronological 70/10/20 splits**; TGN memory + Hawkes history reset at split
  boundaries (enforced by a runtime guard in `SimCityTGN`).
- **Shuffled-timestamp control**: `train.py` re-evaluates with permuted test
  times; Hawkes NLL inflates 15–25×, confirming genuine temporal-order
  learning and no ordering leakage.
- **Causality in the transfer metric**: bridge scores use only events *before*
  a narrative's first platform switch.
- **Confound controls**: popularity (event-count strata) and latent reach
  strata are reported alongside every transfer AUC.
